# Phase 2A Extended: Llama-70B Context Ceiling Test

**Date:** 2026-04-23  
**Model:** huihui-ai/Llama-3.3-70B-Instruct-abliterated  
**Hardware:** 4×H200 SXM (575 GB VRAM), TP=4  
**vLLM:** 0.19.1, prefix caching enabled, chunked prefill, 131K context  

## Purpose

Push Llama-70B to its maximum 131K context window to find the accuracy degradation ceiling. Phase 2A (initial) showed 100% at 14K tokens — this test pushes to 32K, 65K, 100K, and 120K.

## Key Finding: No Degradation Ceiling Found

**100% accuracy from 3K tokens all the way to 91K prompt tokens** — the practical limit of Llama-70B's 131K context window (system prompt + query overhead consumes the remaining ~40K).

### Latency Scaling

Latency is remarkably stable across corpus sizes:
- 24K tokens: 1,219ms avg
- 49K tokens: 1,493ms avg  
- 76K tokens: 1,380ms avg
- 91K tokens: 1,494ms avg

The prefix cache eliminates recomputation of cached tokens. First query at each tier shows higher latency (3-4.5s) as the prefix is computed and cached, but subsequent queries benefit from the cache.

### Throughput Scaling

Throughput decreases with context size due to increased attention computation per query:
- 24K: 2.45/s (at concurrency=3)
- 49K: 1.33/s (at concurrency=2)
- 76K: 0.72/s (at concurrency=1)
- 91K: 0.67/s (at concurrency=1)

Even at 91K tokens, generating a judgment every 1.5 seconds is viable for batch Flock processing.

## Corpus Generation Method

Corpora were generated by combining:
1. 25 real pharmacology findings (base)
2. Structured paragraph-style expansions using 40 domains × 30 compounds × 26 mechanisms × 12 clinical observation types

Each expansion is a realistic-looking finding paragraph (~200 tokens) covering compound interactions, temporal patterns, and pharmacogenomic variables. This tests whether the model can distinguish relevant real findings from dense filler text.

## Implications

1. **Llama-70B's context window is fully usable** — no accuracy penalty at any corpus size
2. **The degradation ceiling, if it exists, is beyond 131K tokens** — need a 1M+ context model to find it
3. **Kimi-Linear (48B-A3B, 1M context, linear attention)** is the next candidate to push beyond 131K
4. **Prefix caching + 4×H200 = unlimited Flock depth** for corpora up to the model's context limit
